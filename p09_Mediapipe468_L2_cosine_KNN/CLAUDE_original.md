## Project Goal

MediaPipe參考網址: https://ai.google.dev/edge/mediapipe/solutions/vision/face_landmarker,
開啟電腦Web cam,輸入人名後,找出468個3-dimensional face landmarks,有x,y,z軸的資料,
把這些資料交給Random Forest做分類訓練後,再偵測做人臉辨識,查看該人是誰。

## Design Decisions
1. 使用相對座標化： 以「鼻尖（Index 1）」為原點 (0, 0, 0)，計算其他點與鼻尖的相對位移。
2. 要能找出距離比例 (Ratios),不要因為臉離鏡頭近一點（臉變大），座標差值就會變大。模型必須處理這種縮放（Scaling）帶來的干擾。

== 目前實測,有訓練過的人,識別效果都不錯,但是沒訓練過的人,很常被誤認為其他人,而不會認為不認識.有什麼建議的解法?
==========================
 Context

 問題：已訓練者辨識效果好，但陌生人（未訓練者）常被誤認為已知人，而非回傳 "Unknown"。
 根因：目前 Unknown 偵測只有單一閘門——k-NN Cosine 相似度 ≥ 0.75。Cosine
 只衡量「方向對齊」，不衡量「距離遠近」。陌生人的 Z-Score
 向量在高維空間可能偶然與某人方向對齊，但實際上離那個人的訓練群集很遠。
 解法：加入第二道閘門——Per-class 中心距離檢查（等同 Diagonal Mahalanobis
 距離），陌生人與所有已知人的訓練群集「距離」都遠，自然被拒絕。

 ---
 數學直覺

 在 Global Z-Score 空間中（未 L2 正規化），計算 query 到 BestName 質心的 L2 距離，等同於：

 Mahalanobis(x, μ_c) ≈ || (x - μ_c) / σ_global ||₂

 即 Diagonal Mahalanobis（以全局標準差為刻度）。比 full Mahalanobis 穩定，不需要 351×351 協方差矩陣求逆，100
 筆樣本即可運作。

 ---
 雙閘門邏輯

 [閘門 1] k-NN Cosine 投票 → BestSim < 閾值 → Unknown（方向不對）
     ↓ 通過
 [閘門 2] ||xStd - Centroid[BestName]||₂ > AcceptRadius → Unknown（距離太遠）
     ↓ 通過
 FinalName = BestName

 AcceptRadius = max(MeanDist + RadiusMultiplier × StdDist, MIN_RADIUS_FLOOR)
 RadiusMultiplier 由 UI Slider 即時調整，不需要重訓（fit 時只存 MeanDist 和 StdDist，predict 時動態計算）。

=== 後來詢問 
❯ 想問問剛剛的修改,是覺得可能是陌生人的z軸距離太相近,所以導致可能被視為別人的機率變多嗎?

● 不完全是這樣，讓我解釋實際根因。

  ---
  真正的問題根源：Cosine 只量「方向」，不量「遠近」

  想像一個簡化的 2D 例子：

         ↗ Alice 的訓練樣本群 (方向角 45°, 距原點 ~8)
        ↗↗↗

  → 陌生人的向量 (方向角 46°, 距原點 ~80)

  Cosine 相似度計算的是向量夾角。陌生人的向量與 Alice 群的方向幾乎相同（角度差 1°），Cosine 就高達 0.99 —
  即使距離差了十倍遠。

  ---
  為什麼陌生人的 Cosine 可以這麼高？

  Z-Score 標準化消除了「所有人臉都長得像人臉」的干擾，讓每個人的向量指向「與平均臉的偏差方向」。

  問題是：陌生人的偏差方向可能偶然和某個已知人的方向對齊，尤其在 351
  維高維空間中，純粹統計上就容易發生「意外對齊」（維度詛咒的副作用）。

  剛才測試的數字就是這個現象：
  陌生人 Cosine = 0.899（超過 0.75 閾值 → 閘門 1 被騙過）
  陌生人 距離   = 68.59（遠超 Alice 群半徑 8.72 → 閘門 2 成功攔截）

  ---
  與 z 軸（深度特徵）的關係

  z 軸（臉部深度，第 326~351 維）是 351 維特徵向量的一部分，但並不是特別的罪魁禍首。問題出在整個向量層面的 Cosine
  計算邏輯，而非哪個軸特別相似。

  加入距離閘門後，本質是在問：「query 點是否真的落在這個人的訓練樣本群集內？」 — 這比單純問方向對不對強健很多。

❯ 那要做到第2點,如何蒐集Unknown樣本?之前有找幾個人,一個人一個人進來訓練Unknown的樣本,讓Unknown是好幾個人的資料,這樣

● 好主意，這個方法是有效的，但有幾個重要細節需要注意。

  為什麼有用

  k-NN 加入 Unknown 樣本後，投票時就多了一個競爭者：

  沒有 Unknown 樣本：Henry×5  yen×2  → 一定輸出已知人
  有了 Unknown 樣本：Henry×3  yen×1  Unknown×3 → 可能輸出 Unknown

  收集越多元的 Unknown 人（不同性別、臉型、年齡），「Unknown 的地盤」在特徵空間裡就越大，陌生人越容易被票投到
  Unknown。

  ---
  但有一個技術問題（p09 特有）

  目前的 Z-Score 標準化是用全部訓練樣本（含 Unknown）計算 μ 和 σ：

  加入 Unknown 後，μ 和 σ 會被 Unknown 樣本污染
  → 已知人的特徵向量的標準化也會跑掉
  → 可能反而讓已知人辨識變差

  要正確實作，Z-Score 的 μ/σ 應只用已知人的樣本計算，Unknown 樣本只參與 k-NN 的投票，不影響標準化。

  ---
  需要修改 cosine_matcher_np.py 的 fit()

  把 Unknown 從統計量計算中排除：
