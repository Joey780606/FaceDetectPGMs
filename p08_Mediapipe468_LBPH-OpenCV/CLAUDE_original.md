=== 1.Mediapipe在人臉角度轉動時,好像還可以順利找出468點.所以是否在實作時,可以接受較大幅度的人臉轉動?
● 很好的問題。我分幾個層面來回答：

  ---
  MediaPipe 的轉動容忍度

  MediaPipe FaceLandmarker 確實對角度有相當好的容忍度：
  - Roll（臉歪斜）：±45° 都還能準確偵測 468 點
  - Yaw（左右轉）：約 ±45~60°，超過後某一側眼睛被遮住，遮住的點會「猜測」補值，精度下降
  - Pitch（上下仰俯）：約 ±30~40°

  ---
  對 p08 方案 B 的影響

  方案 B 用雙眼像素座標做幾何對齊，這裡有個問題：

  ┌───────────────────────┬──────────────────────────────────────────────────────────────────┐
  │         情境          │                               影響                               │
  ├───────────────────────┼──────────────────────────────────────────────────────────────────┤
  │ Roll 角度大（臉歪斜） │ 眼睛還看得到，幾何對齊仍有效                                     │
  ├───────────────────────┼──────────────────────────────────────────────────────────────────┤
  │ Yaw 角度大（側臉）    │ 一眼可能被遮，眼距縮短，對齊品質下降，裁出的臉看起來和正面差很多 │
  ├───────────────────────┼──────────────────────────────────────────────────────────────────┤
  │ Pitch 角度大（仰俯）  │ 臉型變形，LBPH 拿到的影像和正面差異大                            │
  └───────────────────────┴──────────────────────────────────────────────────────────────────┘

  ---
  實際建議

  學習階段：讓使用者稍微左右上下轉頭（約 ±20~30°），增加樣本多樣性。Refmain.py 的 UI 上也有這句提示：

  ▎ "請慢慢左右、上下轉動頭部，以提升辨識多樣性"

  這樣做 LBPH 訓練到多種角度，辨識時遇到角度變化就不會失敗。

  辨識階段：可以用 IOD（瞳距比例） 當過濾條件——如果偵測到的左右眼距離太短（側臉太嚴重），就跳過這幀不推論，避免用扭曲的臉餵給 LBPH 得到錯誤結果。這與 p07 的
   MIN_IOD_NORM 機制相同，只是 p08 的 IOD 是從像素座標計算，不是歸一化座標。

  ---
  結論

  - MediaPipe 本身處理角度是沒問題的
  - 瓶頸在 LBPH：它是影像紋理比對，臉部角度差異大就像是「另一張臉」
  - 解法是：學習時多角度收集樣本 + 推論時過濾極端側臉

  你認為這樣的設計方向可以接受嗎？確認的話，我就按這個邏輯開始實作。
  
=== 2. 如果想要讓程式能接受人臉較大帳度的轉動,有什麼較好輔助方法?
● 這是個很有價值的問題。以下是幾種常見方法，從簡單到複雜排列：

  ---
  方法一：5 點幾何對齊到正臉模板（推薦）

  原理：
  從 468 個 landmark 中選出 5 個穩定關鍵點（左眼中心、右眼中心、鼻尖、左嘴角、右嘴角），計算一個仿射變換（Affine Transform），把這 5
  點對齊到一個預設的正臉模板座標，再裁切固定尺寸影像。

  當前幀的 5 點位置  →  cv2.estimateAffinePartial2D()  →  正臉模板 5 點
                      →  cv2.warpAffine(face_region)
                      →  固定尺寸對齊影像  →  LBPH

  效果：
  - Roll（歪斜）：完全校正
  - Yaw（左右轉）：中等程度校正，轉角越大效果越有限
  - Pitch（仰俯）：部分校正

  優點： 實作簡單，ArcFace、InsightFace 等主流人臉辨識系統都採用這個前處理方式。
  缺點： 對大 Yaw 角（> 45°）效果有限，因為只是 2D 仿射變換，無法真正還原 3D 透視變形。

  ---
  方法二：3D 頭部姿態估計 + 透視校正（進階）

  原理：
  MediaPipe 已經回傳了 3D 歸一化座標（含 z 軸深度資訊）。
  利用這些 3D 點 + cv2.solvePnP 計算出頭部的旋轉矩陣（R）與平移向量（t），再做逆旋轉把臉部透視投影回正面。

  MediaPipe 3D landmarks  →  solvePnP  →  旋轉矩陣 R, 平移 t
                          →  逆旋轉 + 透視 warp
                          →  接近正面的臉  →  LBPH

  效果： 理論上可以處理 Yaw ±60° 以內的大角度轉動。

  優點： 最精確的幾何校正，真正還原 3D 旋轉造成的透視變形。
  缺點： 實作複雜，需要一組 3D 標準臉部模型座標做參考；極大角度時遮擋部位仍是盲點。

  ---
  方法三：角度分 Bin + 多個 LBPH 模型（實用折衷）

  原理：
  從左右眼的 z 座標差異或 IOD 像素比例，估計出 Yaw 角度，分成幾個區間（Bin）：

  Yaw < -30°  →  左側臉 LBPH 模型
  -30° ~ 30°  →  正臉 LBPH 模型
  Yaw > 30°   →  右側臉 LBPH 模型

  訓練時依角度分類存入對應模型，辨識時選對應模型預測。

  優點： 不需要幾何 warp，實作直觀。
  缺點： 樣本需求更多（每個 Bin 都要學足夠的樣本）；Bin 邊界附近容易誤判。

  ---
  方法四：增加訓練樣本多樣性（最簡單）

  學習時讓使用者慢慢轉頭一圈，收集各角度的影像，靠 LBPH 自然學到多角度外觀。
  不需要任何幾何處理，但辨識率提升有限，且訓練 frame 數要拉高（例如從 30 提高到 60~90）。

  ---
  建議

  ┌────────────────────┬──────┬──────┬─────────────────────────────┐
  │        方法        │ 難度 │ 效果 │          適合場景           │
  ├────────────────────┼──────┼──────┼─────────────────────────────┤
  │ 方法一：5 點對齊   │ 低   │ 中高 │ p08 首選，與方案 B 完美契合 │
  ├────────────────────┼──────┼──────┼─────────────────────────────┤
  │ 方法二：solvePnP   │ 高   │ 最好 │ 需要高精度、大角度支援      │
  ├────────────────────┼──────┼──────┼─────────────────────────────┤
  │ 方法三：角度分 Bin │ 中   │ 中   │ 想簡單擴展模型              │
  ├────────────────────┼──────┼──────┼─────────────────────────────┤
  │ 方法四：多樣本     │ 最低 │ 低中 │ 快速驗證用                  │
  └────────────────────┴──────┴──────┴─────────────────────────────┘

  我建議在 p08 採用方法一（5 點對齊），因為：
  1. MediaPipe 已經給了非常精確的 5 個關鍵點
  2. 就算是現在的對齊邏輯，也只是從「只用雙眼」升級成「用 5 點」，改動很小
  3. 效果顯著提升，特別是對 Yaw 和 Pitch 的容忍度
  
=== 3. 目前實測出來,確實光源大小會影響到辨識的結果,關於這部分,有什麼建議嗎?

● 這是 LBPH 的經典問題。LBP
  本身是相對比較（像素與鄰居比大小），理論上有一定的光源不變性，但光源差異太大（整體過亮/過暗、或單側打光）仍會讓直方圖分佈偏移，影響辨識。

  ---
  根本原因

  訓練時：均勻正常光 → LBPH 學到某種直方圖分佈
  推論時：強光/陰影   → 直方圖分佈改變 → 距離變大 → 容易判成 Unknown

  ---
  解決方向

  方法一：CLAHE（推薦，優先做）

  Contrast Limited Adaptive Histogram Equalization
  - 把影像分成小格（8×8），每格獨立做直方圖均衡
  - 比全域 equalizeHist() 更好：能處理單側打光的不均勻情況

  Clahe = cv2.createCLAHE(clipLimit=2.0, tileGridSize=(8, 8))
  Gray  = Clahe.apply(Gray)   # 在 alignFace() 轉灰階後加這一行

  優點： 一行程式碼，效果顯著，是業界最常用的前處理。

  ---
  方法二：Tan & Triggs 正規化（效果最好，稍複雜）

  這是人臉辨識領域專門設計的光源正規化方法，分三步：

  Step 1：Gamma 校正（壓縮高亮區域）
    I_gamma = I ^ 0.2

  Step 2：DoG 濾波（Difference of Gaussians，移除低頻光源分量）
    I_dog = GaussianBlur(sigma=1) - GaussianBlur(sigma=2)

  Step 3：對比度均衡（Contrast Equalization）
    先用 tanh 限幅，再做 L2 正規化

  效果最好，能處理各種光源情境，但程式碼較多。

  ---
  方法三：訓練時多光源採樣（最簡單，效果有限）

  學習階段讓使用者在不同光源條件下各收集一輪（正面光、側面光、昏暗環境），讓 LBPH 學到多種光源下的外觀。
  缺點：每次環境不同都要重新學習，不實用。

  ---
  建議做法

  短期：加 CLAHE，一行程式碼，在 lbph_recognizer.py 的 alignFace() 最後加入，學習和推論都同時處理，光源容忍度明顯提升。

  如果 CLAHE 還不夠： 再考慮 Tan & Triggs。

  要我現在把 CLAHE 加進 lbph_recognizer.py 嗎？改動很小，只有 alignFace() 最後幾行需要修改。

  - 來源：從 Google MediaPipe 官網下載的預訓練模型

  - 來源：由 LbphRecognizer.write() 產生
  - 功能：儲存 LBPH 訓練完成後的模型（OpenCV XML/YAML 格式），包含每個人臉的 LBP 直方圖資料
  - 類比：相當於 p07 的 face_model_rf.pkl（Random Forest 模型檔）
  - 注意：加了 CLAHE 後需要重新訓練覆蓋此檔

  ---
  face_model_lbph_meta.npz
  - 來源：由 face_recognizer.py 的 SaveModel() 產生
  - 功能：儲存兩樣東西：
    a. label index ↔ 人名對應表（例如 0 → "Joey", 1 → "Alice"）
    b. 每個人的對齊臉部影像（100×100 灰階，供 RemovePerson() 重新訓練用）
  - 類比：相當於 p07 的 face_model_rf.pkl 裡面額外存的 label 對應資訊
  - 注意：加了 CLAHE 後需要重新訓練覆蓋此檔
  
=== 5.LBPH 距離是什麼
  LBPH 把每張對齊臉部影像轉換成一個「直方圖向量」，距離就是偵測到的臉的直方圖 和 模型中每個人的直方圖
  之間的差異值：

  距離越小 → 越像訓練過的那個人
  距離越大 → 越不像

  ---
  閾值的本質是取捨

  ┌──────────────────┬──────────────────────────┐
  │       情況       │           說明           │
  ├──────────────────┼──────────────────────────┤
  │ 距離 ≤ threshold │ 判定為已知人（回傳名字） │
  ├──────────────────┼──────────────────────────┤
  │ 距離 > threshold │ 判定為 Unknown           │
  └──────────────────┴──────────────────────────┘

  閾值是一條線，線的兩側都有風險：

        太嚴（低閾值）         太鬆（高閾值）
        ↓                      ↓
  已知人被判成 Unknown    陌生人被判成已知人
  （漏判）                （誤判）

  ---
  120 會不會誤判？

  有可能，但取決於你的資料。

  關鍵在於：陌生人的距離落在哪裡？

  - 如果陌生人的距離都在 150 以上 → 120 安全
  - 如果陌生人的距離落在 100~130 → 120 就會誤判

  你可以直接觀察 console 的 log：

  [LbphRecognizer] 距離=95.3(閾120) → Label=0 Conf=0.60   ← 已知人距離
  [LbphRecognizer] 距離=∞(閾120) → Unknown                 ← 陌生人（完全不像）

  ---
  建議做法

  讓你自己和一個沒有學過的陌生人都對著鏡頭，觀察距離：

  你自己   → 距離應該在 XX 以下
  陌生人   → 距離應該在 XX 以上

  找到兩個範圍之間的「空隙」，把閾值設在空隙中間。這才是最準確的設定方式，而不是靠猜測。
