# p10 人臉辨識方法說明

MediaPipe 468點 + 臉部比例特徵 + Z-Score 標準化 + 線性 SVM 分類

---

## 一、整體流程概覽

```
Webcam 影像
    │
    ▼
MediaPipe FaceLandmarker
    │  偵測人臉並輸出 468 個 3D 座標點
    ▼
特徵萃取（face_feature_3d.py）
    │  從 468 點中選 26 個關鍵點，計算 351 維臉部比例特徵向量
    ▼
Z-Score 標準化 + L2 正規化（svm_classifier_np.py）
    │  消除「人類臉部拓撲相似」的干擾，突顯個人差異
    ▼
線性 SVM 分類（One-vs-Rest）/ 最大 Cosine 相似度（單人模式）
    │  多人：OvR SGD 訓練，取分數最高類別
    │  單人：與所有訓練向量比較，取最大 Cosine 值
    ▼
判斷結果：此人 / Unknown
```

## 二、特徵萃取（351 維）

### 2.1 為何不用全部 468 點

最初嘗試使用全部 468 個 landmark 的 (x, y, z) 座標，攤平成 1404 維向量。
然而測試發現：**任何兩個人的 Cosine 相似度都在 0.997 以上，完全無法區分**。

原因：MediaPipe 468 點描述的是人類臉部的幾何拓撲結構。由於人臉的結構高度保守
（眼睛在鼻子上方、嘴巴在鼻子下方），1404 維向量的「方向」對所有人類幾乎相同，
Cosine 相似度量測的是向量角度，結果永遠趨近於 1。

### 2.2 改用臉部比例特徵（Facial Anthropometry）

選取 26 個關鍵 Landmark，涵蓋臉部主要區域：

| 區域 | Landmark 索引 | 說明 |
|------|-------------|------|
| 鼻子 | 1, 94, 129, 358 | 鼻尖、鼻基底、左右鼻翼 |
| 眼睛 | 33, 133, 159, 145, 263, 362, 386, 374 | 左右眼四角、上下緣 |
| 眉毛 | 46, 105, 107, 276, 334, 336 | 左右眉外緣、頂點、內緣 |
| 嘴巴 | 61, 291, 13, 14 | 左右嘴角、上下唇中心 |
| 下巴 | 152 | 下巴最低點 |
| 顴骨 | 234, 454 | 左右顴骨 |
| 額頭 | 10 | 額頭中心 |

### 2.3 特徵向量組成（351 維）

**Part A：成對 3D 距離（325 維）**

計算 26 個關鍵點之間所有可能的成對 3D 距離，共 C(26, 2) = 325 對。
每個距離除以 IOD（瞳距）做尺度歸一化，消除臉離鏡頭遠近的影響。

```
距離(點i, 點j) / IOD   → 無論臉大臉小，比值相同
```

**Part B：z 深度（26 維）**

26 個關鍵點各自相對鼻尖的 z 軸深度，除以 IOD。
編碼臉部的立體結構，例如鼻子的高挺程度、眼窩的深淺。

**最終向量：325 + 26 = 351 維**

### 2.4 IOD 歸一化

IOD（Inter-Ocular Distance，瞳距）= 左眼中心與右眼中心的 3D 距離。

```
左眼中心 = 平均(Landmark 33, 159, 145, 133)
右眼中心 = 平均(Landmark 362, 386, 374, 263)
IOD = 3D 距離(左眼中心, 右眼中心)
```

所有距離與深度值均除以 IOD，確保縮放不變性。
若 IOD < 1e-5（MediaPipe 偵測退化），丟棄此幀。

## 三、辨識引擎（Z-Score 標準化 + 線性 SVM）

### 3.1 為何需要 Z-Score 標準化

即使改用 351 維比例特徵，直接計算 Cosine 相似度仍然失敗（相似度 ~0.997）。

原因：351 個特徵值都是正數（距離 / IOD），且對所有人類都落在相近的數值範圍，
351 維向量仍然指向「正數象限」的相似方向，角度差異極小。

**解法：Z-Score 標準化**

```
標準化後的特徵 = (原始值 - 所有訓練樣本該維度的平均) / 標準差

意義轉換：
  原本：「我的鼻寬 / 瞳距 = 0.82」         ← 任何人都差不多
  標準化後：「我的鼻寬比訓練樣本平均高 2.3 個標準差」 ← 每個人不同
```

標準化後，向量描述的是「這張臉與平均臉的偏差方向」，不同人的偏差方向不同。

### 3.2 多人模式（≥ 2 人）：One-vs-Rest 線性 SVM

**訓練階段：**

```
1. 收集所有人的所有訓練樣本（100 幀 × N 人）
2. 計算每個特徵維度的全局均值 μ[d] 與標準差 σ[d]
   （標準差為 0 的維度設為 1，代表無辨別力）
3. 對所有樣本做 Z-Score 標準化：x'[d] = (x[d] - μ[d]) / σ[d]
4. 對標準化後的向量做 L2 正規化（使長度為 1）
5. OvR：對每個人 i 訓練一個二元 SVM
   正類（+1）= 此人的所有樣本
   負類（-1）= 其他所有人的樣本
   使用 mini-batch SGD + hinge loss + L2 正則化（λ=0.001）
6. 儲存權重矩陣 W（N_classes × 351）與偏置 b（N_classes,）
```

**推論階段：**

```
1. 對 query 做相同的 Z-Score 標準化 + L2 正規化
2. 計算各類別原始分數：Scores = W @ xn + b
3. 取分數最高的類別為獲勝者
4. 轉換為信心度：Conf = sigmoid(max_score) = 1 / (1 + exp(-score))
5. Conf ≥ 閾值 → 辨識為此人；否則 → Unknown
```

### 3.3 單人模式（僅 1 人）：最大 Cosine 相似度

僅有一位訓練者時，無法對「其他所有人」建立負類邊界。
改用最大 Cosine 相似度比對，邏輯同 p09 k-NN：

```
1. 對 query 做 Z-Score + L2 正規化
2. 計算與所有訓練向量的 Cosine 相似度（矩陣乘法）
3. 取最大相似度，轉換為 sigmoid 信心度
4. Conf ≥ 閾值 → 辨識為此人；否則 → Unknown
```

### 3.4 Unknown 偵測

不需要事先收集陌生人樣本。
陌生人的 Z-Score 偏差方向與已訓練者不同，分類器輸出分數低，
sigmoid 轉換後低於閾值 → Unknown。

### 3.5 SVM 超參數

| 參數 | 預設值 | 說明 |
|------|--------|------|
| λ（L2 正則化） | 0.001 | C = 500，大邊距，適合 100 張 webcam 樣本 |
| epochs | 200 | 對 2-10 人資料集 < 0.1 秒 |
| learning_rate | 0.01 | 初始學習率 |
| lr_decay | 0.99 | 每 epoch 衰減，避免震盪 |
| batch_size | 16 | mini-batch SGD |

### 3.6 信心度閾值調整

UI 提供「SVM 信心度閾值」Slider（範圍 0.10 ~ 0.99，預設 0.60）：

| sigmoid 值 | 對應原始分數 | 意義 |
|-----------|------------|------|
| 0.99 | +4.6 | 非常嚴格，接近確定無誤 |
| 0.75 | +1.1 | 嚴格，距離決策邊界有一段距離 |
| 0.60 | +0.4 | 預設，略超過決策邊界即接受 |
| 0.50 | 0.0 | 恰好在決策邊界上 |

### 3.7 動態閾值補償（頭部姿態）

**問題根因：** 訓練時側臉/仰頭/低頭的樣本數少於正臉，導致 Z-Score 均值偏向正臉分布，
SVM 決策邊界對非正臉姿態的評分偏低，容易誤判為 Unknown。

**解法：** 推論時估算每張臉的水平（Yaw）與垂直（Pitch）轉角比例，
動態降低該幀的有效信心度閾值：

```
effective_threshold = max(0.10, base_threshold - min(0.20, combined_ratio × 0.30))
combined_ratio = √(YawRatio² + PitchRatio²)
```

**Yaw 估算（水平轉角，`_computeYawRatio`）：**

```
LeftCheekX  = Landmark[234].x
RightCheekX = Landmark[454].x
NoseTipX    = Landmark[1].x
FaceWidth   = max(LeftCheekX, RightCheekX) - min(...)
YawRatio    = |DistA - DistB| / FaceWidth   # DistA/B 為鼻尖到兩側距離
```

正臉時鼻尖居中 → YawRatio ≈ 0；側臉 → YawRatio → 1。

**Pitch 估算（垂直傾角，`_computePitchRatio`）：**

```
EyeY       = 平均(Landmark[33, 133, 159, 145, 263, 362, 386, 374].y)
ForeheadY  = Landmark[10].y
ChinY      = Landmark[152].y
EyeRatio   = (EyeY - ForeheadY) / (ChinY - ForeheadY)
PitchRatio = min(1.0, |EyeRatio - 0.42| / 0.15)
```

正臉時眼睛約在臉部高度 42% 處 → PitchRatio ≈ 0；抬/低頭 → PitchRatio → 1。

**補償效果：**

| combined_ratio | 補償量 | 有效閾值（基準 0.60） |
|---------------|--------|----------------------|
| 0.00（正臉）  | 0.00   | 0.60 |
| 0.33（輕微）  | 0.10   | 0.50 |
| 0.67（中等）  | 0.20   | 0.40 |
| ≥ 0.67        | 最多 0.20 | 最低 0.10 |
